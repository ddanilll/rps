# Сетевой калькулятор с переменными X, Y, Z

Лабораторная работа: TCP клиент-серверное приложение с неблокирующим вводом-выводом и `poll(2)`.

## Что реализовано

Сервер хранит для каждого пользователя три переменные `X`, `Y`, `Z`.
Клиент подключается к серверу, отправляет имя пользователя, затем команды:

```text
X = 10 + 5
Y = X * 2
вычислить Y / 3
показать
```

Поддерживаются:

- целые числа;
- переменные `X`, `Y`, `Z`;
- операции `+`, `-`, `*`, `/`;
- скобки;
- унарный минус, например `X = -5`.

Сервер проверяет корректность выражения, вычисляет результат и отвечает числом
или понятным сообщением об ошибке.

Значения переменных сохраняются для каждого пользователя в файле `calc_users.db`,
поэтому после переподключения пользователь видит свои старые значения.

## Сетевой протокол

Протокол текстовый, поверх TCP. Одно сообщение — одна строка, завершенная `\n`.

### Подключение

1. Клиент открывает TCP-соединение.
2. Сервер отправляет:
   ```text
   HELLO enter username
   ```
3. Клиент отправляет имя пользователя:
   ```text
   ivan
   ```
4. Сервер отвечает:
   ```text
   OK user ivan X=0 Y=0 Z=0
   ```

### Команды пользователя

#### Присваивание

```text
X = 2 + 3 * 4
```

Ответ:

```text
OK X = 14
```

#### Вычисление без изменения переменных

```text
вычислить X + Y / 2
```

Ответ:

```text
OK 20
```

Также поддерживается английский вариант:

```text
calc X + Y / 2
```

#### Показать переменные

```text
показать
```

Ответ:

```text
OK X=14 Y=0 Z=0
```

Также поддерживается:

```text
show
```

#### Выход

```text
выход
```

или

```text
quit
```

### Ошибки

Примеры ответов:

```text
ERR неизвестная команда
ERR деление на ноль
ERR ожидался операнд
ERR неизвестная переменная: A
```

### Режим тестирования производительности

Для измерения сетевой части без вычислений предусмотрена команда:

```text
PING
```

Сервер отвечает:

```text
PONG
```

Эта команда не запускает парсер выражений и не меняет переменные.

Клиентский режим нагрузки:

```bash
./netcalc client 127.0.0.1 4321 --perf 100000
```

Клиент отправит 100000 команд `PING` и покажет время, запросы в секунду и среднюю задержку.

## Сборка

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

## Запуск

В одном терминале:

```bash
./netcalc server 4321
```

В другом терминале:

```bash
./netcalc client 127.0.0.1 4321
```

## Архитектура

Программа состоит из четырех основных частей:

- `main.cpp` — разбор аргументов командной строки;
- `server.cpp` — TCP-сервер, неблокирующие сокеты, `poll`;
- `client.cpp` — консольный клиент;
- `expression.cpp` — парсер и вычислитель арифметических выражений.

Сервер является однопроцессным и однопоточным. Он не создает процесс или поток
на каждого клиента. Вместо этого все сокеты переводятся в неблокирующий режим,
а системный вызов `poll(2)` сообщает, какой сокет сейчас готов к чтению или записи.

