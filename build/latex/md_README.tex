Лабораторная работа\+: TCP клиент-\/серверное приложение с неблокирующим вводом-\/выводом и {\ttfamily poll(2)}.\hypertarget{md_README_autotoc_md1}{}\doxysection{Что реализовано}\label{md_README_autotoc_md1}
Сервер хранит для каждого пользователя три переменные {\ttfamily X}, {\ttfamily Y}, {\ttfamily Z}. Клиент подключается к серверу, отправляет имя пользователя, затем команды\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{X = 10 + 5}
\DoxyCodeLine{Y = X * 2}
\DoxyCodeLine{вычислить Y / 3}
\DoxyCodeLine{показать}

\end{DoxyCode}


Поддерживаются\+:


\begin{DoxyItemize}
\item целые числа;
\item переменные {\ttfamily X}, {\ttfamily Y}, {\ttfamily Z};
\item операции {\ttfamily +}, {\ttfamily -\/}, {\ttfamily $\ast$}, {\ttfamily /};
\item скобки;
\item унарный минус, например {\ttfamily X = -\/5}.
\end{DoxyItemize}

Сервер проверяет корректность выражения, вычисляет результат и отвечает числом или понятным сообщением об ошибке.

Значения переменных сохраняются для каждого пользователя в файле {\ttfamily calc\+\_\+users.\+db}, поэтому после переподключения пользователь видит свои старые значения.\hypertarget{md_README_autotoc_md2}{}\doxysection{Сетевой протокол}\label{md_README_autotoc_md2}

\hypertarget{md_README_autotoc_md13}{}\doxysection{Архитектура}\label{md_README_autotoc_md13}
Программа состоит из четырех основных частей\+:


\begin{DoxyItemize}
\item {\ttfamily main.\+cpp} — разбор аргументов командной строки;
\item {\ttfamily server.\+cpp} — TCP-\/сервер, неблокирующие сокеты, {\ttfamily poll};
\item {\ttfamily client.\+cpp} — консольный клиент;
\item {\ttfamily expression.\+cpp} — парсер и вычислитель арифметических выражений.
\end{DoxyItemize}

Сервер является однопроцессным и однопоточным. Он не создает процесс или поток на каждого клиента. Вместо этого все сокеты переводятся в неблокирующий режим, а системный вызов {\ttfamily poll(2)} сообщает, какой сокет сейчас готов к чтению или записи.\hypertarget{md_README_autotoc_md14}{}\doxysection{Пример работы}\label{md_README_autotoc_md14}
